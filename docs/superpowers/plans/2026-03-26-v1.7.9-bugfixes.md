## What's Deferred (documented, not in v1.7.9)

These bugs require the metadata architecture redesign (documented in `docs/architecture/metadata-redesign.md`). They will be addressed in a dedicated initiative:

| Bug | Root Cause | Fix Approach |
|-----|-----------|--------------|
| VA albums split by artist | No `album_artist` field; library groups by per-track artist | New `albums` + `album_tracks` + `track_files` schema |
| Cross-album track bleed | Library view merges tracks from all albums by fuzzy artist match | Hard FK `album_tracks.album_id` with exact matching |
| Duplicate tracks (same title, different YT source) | No canonical track definition; each download creates its own entry | One `album_tracks` row per MB track, `track_files` is 1:1 |
| [unknown] track titles | Titles derived from YT filename, not MusicBrainz | Titles locked at album creation from MB metadata |
| Duration mismatch (MB vs YT) | Duration calculated from file, not stored from MB | `album_tracks.duration` from MB, `track_files.file_duration` separate |
| Recently played duplicates per track artist | Recently played keyed by track artist, not album artist | Use `albums.album_artist` for grouping |
| Underscore in album name | Colon sanitized to underscore in filesystem path | Sanitization should use dash or remove, not underscore |
| Rapid skip (4 clicks = 1 advance) | pendingIdxRef not working in deployed build | Investigate — may be React concurrent mode or ref timing issue |
| Simplistic duplicate (1 track shows as 2) | Library scan creates duplicate entries | Dedup by stable track ID from MB |
